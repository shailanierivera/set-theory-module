Step-by-Step Walkthrough
Classifying Functions: Is it Injective, Surjective, or Bijective?
Step 1 of 5
🎯 The Problem
Let f: {1,2,3} → {a,b,c} with f(1)=a, f(2)=b, f(3)=c.
Let g: {1,2,3} → {a,b} with g(1)=a, g(2)=b, g(3)=a.
Let h: {1,2,3,4} → {a,b,c} with h(1)=a, h(2)=b, h(3)=c, h(4)=a.

We will classify each as injective, surjective, or bijective.
Step 2 of 5
🔍 Test f for Injectivity
Injectivity means: no two different inputs share the same output.
f(1)=a, f(2)=b, f(3)=c All outputs are DIFFERENT → f is INJECTIVE ✅ Contrapositive check: a₁≠a₂ ⟹ f(a₁)≠f(a₂) 1≠2: f(1)=a ≠ b=f(2) ✓ 1≠3: f(1)=a ≠ c=f(3) ✓ 2≠3: f(2)=b ≠ c=f(3) ✓
Step 3 of 5
🔍 Test f for Surjectivity
Surjectivity means: every element of the codomain is hit.
Codomain = {a, b, c} a is hit by 1 ✓ b is hit by 2 ✓ c is hit by 3 ✓ Every element is hit → f is SURJECTIVE ✅ f is BIJECTIVE (injective AND surjective) ✅
Step 4 of 5
🔍 Classify g: {1,2,3} → {a,b}
g(1)=a, g(2)=b, g(3)=a
Injectivity: g(1)=g(3)=a but 1≠3 → NOT injective ❌ Surjectivity: a is hit by 1 (and 3) ✓ b is hit by 2 ✓ All codomain elements hit → SURJECTIVE ✅ g is SURJECTIVE ONLY (not injective, not bijective)
Step 5 of 5
🔍 Classify h: {1,2,3,4} → {a,b,c}
h(1)=a, h(2)=b, h(3)=c, h(4)=a
Injectivity: h(1)=h(4)=a but 1≠4 → NOT injective ❌ Surjectivity: a is hit by 1,4 ✓ b is hit by 2 ✓ c is hit by 3 ✓ All codomain elements hit → SURJECTIVE ✅ Summary: f → BIJECTIVE (injective + surjective) g → SURJECTIVE (onto but not one-to-one) h → SURJECTIVE (onto but not one-to-one) Key insight: when |domain| > |codomain|, injectivity is impossible. When |domain| < |codomain|, surjectivity is impossible.

Step-by-Step Walkthrough
Cantor's Diagonal Argument — Why ℝ is Uncountable
Step 1 of 4
Assume a complete list exists
We proceed by contradiction. Suppose all real numbers in [0,1] can be listed as r₁, r₂, r₃, … We write each in decimal form.
r₁ = 0.d₁₁ d₁₂ d₁₃ d₁₄ ... r₂ = 0.d₂₁ d₂₂ d₂₃ d₂₄ ... r₃ = 0.d₃₁ d₃₂ d₃₃ d₃₄ ... r₄ = 0.d₄₁ d₄₂ d₄₃ d₄₄ ... ...
Step 2 of 4
Build the diagonal number
Construct a number d by looking at the diagonal digit dₙₙ and changing it: if dₙₙ ≠ 5, use 5; if dₙₙ = 5, use 4.
d = 0.e₁ e₂ e₃ e₄ ... where eₙ ≠ dₙₙ for every n Example: if diagonal is 3,7,1,5,... Then d = 0.5 5 5 4 ... (5 when diagonal≠5, else 4)
Step 3 of 4
d is not on the list
Why is d not equal to any rₙ in our list?
Is d = r₁? NO — d differs from r₁ in position 1 (e₁ ≠ d₁₁) Is d = r₂? NO — d differs from r₂ in position 2 (e₂ ≠ d₂₂) Is d = rₙ? NO — d differs from rₙ in position n (eₙ ≠ dₙₙ) For EVERY n: d ≠ rₙ Therefore d is not anywhere on our "complete" list!
Step 4 of 4
Contradiction — ℝ is uncountable
We assumed our list was complete, but we just found a real number d ∈ [0,1] that isn't on it. Contradiction!
Conclusion: No complete list of real numbers can exist. Therefore ℝ is UNCOUNTABLE. |ℝ| = 𝔠 = 2^{ℵ₀} > ℵ₀ = |ℕ| This shows there are genuinely different sizes of infinity — the real numbers are a "bigger" infinity than the naturals.

Step-by-Step Walkthrough
Constructing ℤ from ℕ — A Complete Example
Step 1 of 5
The Problem with ℕ
The natural numbers ℕ = {0,1,2,3,…} cannot solve equations like x + 5 = 2. We need negative numbers. How do we build them from sets alone?
Goal: Construct ℤ = {…,-2,-1,0,1,2,…} Starting from: ℕ = {0,1,2,3,…} Method: Encode an integer as a DIFFERENCE of two naturals
Step 2 of 5
Form ℕ × ℕ
Consider all ordered pairs (a,b) with a,b ∈ ℕ. Think of (a,b) as representing "a − b".
Examples: (5, 2) represents 5 − 2 = +3 (2, 7) represents 2 − 7 = −5 (4, 4) represents 4 − 4 = 0 Problem: Many pairs represent the SAME integer: (3,0), (4,1), (5,2) all represent +3 We need to identify them!
Step 3 of 5
Define the Equivalence Relation
Two pairs represent the same integer when their "differences" are equal: a−b = c−d, which (avoiding subtraction!) means a+d = b+c.
Define: (a,b) ~ (c,d) iff a + d = b + c Examples: (5,2) ~ (6,3)? 5+3 = 2+6 → 8=8 ✓ (both = +3) (5,2) ~ (3,0)? 5+0 = 2+3 → 5=5 ✓ (both = +3) (5,2) ~ (1,4)? 5+4 = 2+1 → 9≠3 ✗ (not same integer)
Step 4 of 5
Form the Quotient Set
The integers ARE the equivalence classes of (ℕ×ℕ)/~. Each class is one integer.
+3 = [(3,0)] = {(3,0),(4,1),(5,2),(6,3),…} −2 = [(0,2)] = {(0,2),(1,3),(2,4),(3,5),…} 0 = [(0,0)] = {(0,0),(1,1),(2,2),(3,3),…} +1 = [(1,0)] = {(1,0),(2,1),(3,2),(4,3),…} These classes partition ℕ×ℕ completely.
Step 5 of 5
Arithmetic on ℤ
We define + and × on these classes. We must verify these are "well-defined" (same result regardless of which representative we pick).
Addition: [(a,b)] + [(c,d)] = [(a+c, b+d)] Negation: −[(a,b)] = [(b,a)] ← swap! Multiplication: [(a,b)]·[(c,d)] = [(ac+bd, ad+bc)] Example: (+3) + (−2) = [(3,0)] + [(0,2)] = [(3+0, 0+2)] = [(3,2)] = +1 ✓ This is a complete, rigorous construction of ℤ from sets!

Step-by-Step Walkthrough
Transfinite Induction — A Complete Proof
Step 1 of 5
The Claim
We prove by transfinite induction that every ordinal α satisfies P(α): "α is a transitive set", meaning if β ∈ γ ∈ α then β ∈ α.
Transitive set: if something belongs to an element of α, then it belongs to α directly. Examples: α = {0,1,2} = 3. Is 0 ∈ 1 ∈ 3? → Is 0 ∈ 3? YES ✓ We need to prove this for ALL ordinals, including ω, ω+1, etc.
Step 2 of 5
Case 1 — Base: P(0)
We prove P(0): 0 = ∅ is a transitive set.
0 = ∅ has NO elements. The condition "β ∈ γ ∈ ∅" is vacuously false (nothing belongs to ∅). So the implication β ∈ γ ∈ 0 → β ∈ 0 is vacuously TRUE. ✅ P(0) holds.
Step 3 of 5
Case 2 — Successor: P(β) → P(β+1)
Assume P(β): β is transitive. We prove P(β+1): β+1 = β∪{β} is transitive.
Let γ ∈ δ ∈ β+1 = β∪{β}. Case A: δ = β Then γ ∈ β ⊆ β+1. ✓ Case B: δ ∈ β Then γ ∈ δ ∈ β. By P(β), β is transitive → γ ∈ β ⊆ β+1. ✓ Either way, γ ∈ β+1. So β+1 is transitive. P(β+1) holds. ✅
Step 4 of 5
Case 3 — Limit: P(λ) for limit ordinal λ
Assume P(β) for all β < λ (every β < λ is transitive). Prove P(λ): λ is transitive. Note λ = ∪{β | β < λ}.
Let γ ∈ δ ∈ λ. Since δ ∈ λ = ∪{β | β < λ}, ∃ β₀ < λ such that δ ∈ β₀. By the inductive hypothesis, P(β₀) holds: β₀ is transitive. γ ∈ δ ∈ β₀ and β₀ transitive → γ ∈ β₀. β₀ ⊆ λ, so γ ∈ λ. ✓ P(λ) holds. ✅
Step 5 of 5
Conclusion
All three cases are verified. By transfinite induction, every ordinal is a transitive set.
By transfinite induction: Base case ✓ Successor case ✓ Limit case ✓ ∴ Every ordinal α is a transitive set. □ Key takeaway: The LIMIT CASE is what makes transfinite induction more powerful than ordinary induction — it handles the "jumps" at limit ordinals like ω, ω·2, ε₀. Without it, we could only prove P(n) for finite n.
⚡
